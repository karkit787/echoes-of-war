# Cocos Creator Migration Plan

> Superseded for Chapter 0 runtime placement by
> `docs/dev/chapter0_integration_summary.md`. Chapter 0 is now intentionally
> loaded from `assets/resources/` through its manifest.

**Project:** Echoes of War  
**Cocos version:** 3.8.8  
**Migration date:** 12 June 2026  
**Status:** Implemented and validated

## Goal

Keep the existing Echoes of War vertical slice in a clean Cocos Creator
project without losing authored content or invalidating Cocos UUID metadata.

## Placement Rules

- Runtime files live under `assets/`.
- TypeScript lives under `assets/scripts/`.
- Serialized Cocos scenes live under `assets/scenes/`.
- Files loaded by string through `resources.load()` live under
  `assets/resources/`.
- Planning, writing, and generation provenance live under `content/`.
- Developer, design, curriculum, and reference material live under `docs/`.
- Generated folders such as `build/`, `library/`, `temp/`, and `local/` stay
  untracked.

## Audit Result

The existing files had already been migrated into the intended high-level
layout before this Cocos project was initialized:

| Content | Current location | Loading model | Decision |
|---|---|---|---|
| TypeScript systems | `assets/scripts/` | Imported by components | Keep |
| Chapter 0 dialogue | `assets/dialogue/chapter0/` | Inspector-bound `JsonAsset[]` | Keep |
| Chapter 0 backgrounds | `assets/images/chapter0/backgrounds/` | Inspector-bound `SpriteFrame` registry | Keep |
| Chapter 0 portraits | `assets/images/chapter0/characters/` | Inspector-bound `SpriteFrame` registry | Keep |
| Dialogue demo JSON | `assets/resources/ui/dialogue/` | `resources.load()` | Keep |
| Dialogue UI SVGs | `assets/ui/dialogue/` | Inspector-bound UI assets | Keep |
| Chapter 0 scene shell | `assets/scenes/Chapter0.scene` | Cocos serialized scene | Keep and bind |
| Chapter writing and art prompts | `content/chapters/` | Non-runtime source | Keep |
| Project guidance and references | `docs/` | Non-runtime documentation | Keep |

The Chapter 0 JSON and PNG files are not dynamically loaded by path. Moving
them under `assets/resources/` would increase the resources bundle and would
require moving their generated `.meta` files as inseparable pairs. There is no
runtime benefit at this stage, so they remain editor-bound assets.

## Metadata Safety

Cocos Creator has generated `.meta` files for all current runtime assets and
asset directories. These files contain UUIDs and must be committed alongside
their assets.

Rules for future moves:

1. Prefer moving imported assets through the Cocos Creator Asset panel.
2. If a filesystem move is unavoidable, move the asset and its adjacent
   `.meta` file together.
3. Never regenerate or hand-edit UUIDs to make a move appear successful.
4. Reopen Cocos and check scene/prefab references after any imported asset move.

No imported runtime asset was moved during this migration pass because the
current paths already satisfy the loading model and metadata had been created.

## Validation

Run:

```powershell
powershell -ExecutionPolicy Bypass -File tools/content-validation/validate-cocos-migration.ps1
powershell -ExecutionPolicy Bypass -File tools/content-validation/validate-chapter0-scene.ps1
node "C:\ProgramData\cocos\editors\Creator\3.8.8\resources\app.asar.unpacked\node_modules\typescript\lib\tsc.js" -p tsconfig.json --noEmit
git -c safe.directory=D:/FYP/echoes-of-war-cocos diff --check
```

The migration validator parses runtime/source JSON and Cocos `.meta` files,
checks asset/meta pairing, confirms the dynamic demo resource path, and checks
that generated directories are ignored and untracked.

`tsconfig.json` is JSONC generated by Cocos and intentionally contains
comments. It is validated by TypeScript/Cocos rather than a strict JSON parser.
